# Support Request Workflow

Creators open **Contact an Admin**, choose a category, write a message, preview it, and
confirm delivery. The request is stored before Telegram delivery, so a routing problem
cannot erase it. Creators see only their own requests under **My Support Requests**.

Authorized Admins see open requests in **Support Requests** and Admin Queue. They may assign
a request, open the permitted creator profile, add a private note, escalate it to an Owner,
or resolve it. Assignment, escalation, and resolution are audited.

The centralized routing service delivers to the configured Admin Support topic using both
chat ID and topic ID. Failed delivery creates a safe reference and durable record; private
content is never routed to the Main or Sellers group.
